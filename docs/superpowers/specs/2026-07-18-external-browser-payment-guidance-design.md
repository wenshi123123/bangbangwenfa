# 微信内付款外部浏览器引导

## 目标

在微信内置浏览器中，用户进入任何付款页时不创建支付订单、不显示支付按钮，统一提示用户点击右上角“…”并选择“在浏览器打开”。用户在外部浏览器重新打开相同链接后，继续现有的微信支付流程。

## 覆盖范围

- 咨询付款页：`/pay?orderId=...`
- 律师入驻付款页：`/lawyer/pay?applicationId=...`
- 律师会员续费页：`/lawyer/renew`

## 行为规则

1. 页面加载后检测是否处于微信内置浏览器。
2. 若在微信内：渲染统一的全屏引导页，说明“右上角 … → 在浏览器打开”，并保留当前页面链接供用户复制；不得调用创建支付订单的接口。
3. 若不在微信内：沿用现有支付流程，包括 H5 唤起微信、PC 二维码及已存在的支付结果确认。
4. 已存在的待付款订单在微信内也只显示引导；离开微信后仍使用同一订单链接继续付款。
5. 支付回调、订单状态查询、支付成功页面和现有外部浏览器支付逻辑不修改。

## 实现边界

- 提取可复用的微信内浏览器付款引导组件，三处付款页共同使用。
- 付款页必须在调用创建支付接口之前短路返回引导界面，避免产生未付款支付单。
- 不改微信支付服务端配置、回调地址、订单金额或登录逻辑。

## 验证

- 静态测试：三处付款页均复用引导组件，并在微信内不触发创建支付请求。
- 本地页面验证：模拟/检测微信内环境时显示引导；普通浏览器仍显示正常支付入口。
- 编译验证：生产构建通过。
